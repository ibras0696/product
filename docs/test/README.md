# Исследовательская БД: история Чечни (SQLite)

Назначение: собрать в одну реляционную базу **проверяемые** факты о Чечне —
местности, личности (герои), события (включая войны), подвиги и то, где герои
жили. База — исследовательский seed для наполнения каталога, а не источник
публикации напрямую.

## ⚠️ Достоверность (важно)

Проект запрещает выдумывать исторические факты (BR-008 в [../../TZ.md](../../TZ.md)).
Поэтому:

- каждая фактическая строка несёт `source_id` (ссылку на источник) и
  `verification_status`;
- `verified` — факт вручную сверен с официальным, архивным или иным профильным
  источником; статус относится к конкретной записи, а не автоматически ко всему сайту;
- `corroborated` — запись подтверждается профильным/официальным справочником, но
  ещё не прошла полную музейную редактуру;
- `needs_review` — факт массово найден в открытых источниках и **требует финальной проверки
  владельцем контента перед публикацией**;
- точность координат хранится отдельно от достоверности исторического факта;
  для 443 объектов добавлена независимая точка OpenStreetMap и расстояние до неё.

Ничего из этой базы не должно попадать в публичный каталог без прохождения
обычной модерации с подтверждающим источником.

## Как собрать

```bash
python3 docs/test/build_db.py            # создаёт docs/test/sqlite.db
python3 docs/test/build_db.py --db /tmp/other.db
python3 docs/test/harvest_wikidata.py    # обновляет массовый research-слой
python3 docs/test/harvest_osm.py         # независимая проверка геолокации
python3 docs/test/harvest_commons_categories.py  # дополнительные фото (учитывать API rate limit)
```

Скрипт идемпотентен: пересоздаёт таблицы из `schema.sql` и заполняет их из
ручного `dataset.py`, кураторского `deep_research.py` и массового
`harvested_data.json`, резолвя `slug`/Wikidata QID
в внешние ключи, затем проверяет
целостность (`PRAGMA foreign_key_check`) и печатает счётчики.

`harvest_wikidata.py` получает исследовательские кандидаты из Wikidata и
лицензионные метаданные изображений из Wikimedia Commons. Он не присваивает им
`verified`: автоматический сбор не заменяет проверку источников элемента.

Зависимости: только стандартная библиотека Python 3 (`sqlite3`). Для ручных
запросов удобен CLI `sqlite3`.

## Схема

```
sources ─┬─< places ─┐
         ├─< people ──┤ (birthplace_id → places)
         ├─< events ──┤ (place_id → places)
         └─< deeds     │
districts ──< places ─┘

people ──< deeds >── events
people ──< residences >── places      (где жили: born|lived|worked|died|buried|commemorated)
people ──< person_events >── events   (участие в событиях)
places|people|events ──< media_assets (Commons URL, автор, лицензия)
places ──< coordinate_evidence           (независимые координаты и расхождение)
sources ──< source_documents ──< fact_citations >── places|people|events|deeds
```

Таблицы:

- **sources** — источники истины (title, publisher, url, reliability).
- **districts** — 15 административных районов ЧР.
- **places** — местности: города, сёла, крепости, башни, мечети, озёра, реки,
  горы, музеи, мемориалы (+ несколько внешних мест для биографий).
- **people** — личности: герои ВОВ, деятели Кавказской войны, революционные,
  государственные, религиозные и культурные фигуры.
- **events** — войны, сражения, восстания, депортация, основания, строительство.
- **deeds** — подвиги: конкретные деяния личности (+ награда, событие, место).
- **residences** — связь личность↔место с типом отношения.
- **person_events** — участие личностей в событиях (многие-ко-многим).
- **media_assets** — ссылки на страницы и оригиналы изображений, авторство и лицензии.
- **coordinate_evidence** — независимые точки OSM, метод совпадения и расстояние
  до основной координаты; эти данные не перезаписывают карточку места.
- **source_documents** — конкретные архивные публикации, книги, законы,
  музейные лекции и биографические профили.
- **fact_citations** — связь одного конкретного утверждения с документом и объектом.

Для геолокации `coordinate_accuracy` имеет значения `exact`, `approximate` или
`unknown`; `coordinate_precision_m` хранит заявленную точность, а
`coordinate_source_url` — страницу источника координат. Координаты населённых пунктов
часто являются центром объекта, а не точной границей.

Готовые представления: `v_hero_birthplaces` (герои и места рождения),
`v_place_people` (кто с каким местом связан), `v_fact_evidence` (утверждение,
документ, ссылка и уровень источника).

## Примеры запросов

```sql
-- Герои и где они родились
SELECT * FROM v_hero_birthplaces;

-- Подвиги с наградами и источником
SELECT pe.full_name_ru, d.title, d.award, s.title AS source
FROM deeds d
JOIN people pe ON pe.id = d.person_id
LEFT JOIN sources s ON s.id = d.source_id;

-- Все объекты одного района
SELECT name_ru, place_type FROM places
WHERE district_id = (SELECT id FROM districts WHERE slug = 'nozhay-yurtovsky');

-- Что ещё требует проверки
SELECT 'people' AS t, full_name_ru AS name FROM people WHERE verification_status='needs_review'
UNION ALL
SELECT 'places', name_ru FROM places WHERE verification_status='needs_review';
```

обязательно запускаются сборка и тесты.
